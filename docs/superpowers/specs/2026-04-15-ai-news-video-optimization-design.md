# AI News Video — 优化设计文档

**日期**：2026-04-15  
**状态**：已确认，待实现

---

## 背景

当前 ai-news-video 工具能完成从主题到视频的全流程，但存在以下问题：

1. 生成文案偏"产品说明书"风格，缺乏故事感和情绪
2. 钩子句吸引力不足，观众容易划走
3. TTS 口播节奏感差，句子过长
4. 社媒文案（抖音/小红书）质量参差
5. HTML 分镜全为文字，无真实图片，视觉单调
6. 模板种类少（仅 5 种），无法覆盖多样内容结构
7. 所有视频同一视觉风格，无法适配不同主题调性

---

## 目标

完成一次整体重构，覆盖四个方向：

1. **文案质量提升** — 改进 SKILL.md 口播/社媒写作指令
2. **视觉主题系统** — 4 套主题，LLM 自动推断 + 用户可覆盖
3. **模板库扩充** — 从 5 种扩展到 13 种，含图片类模板
4. **图片采集集成** — Playwright 截图 + 页面图片提取，保存本地后嵌入 HTML

---

## 架构变化

### 新流程

```
新闻主题
  → [新增] 搜索 + 截图/抓图 → output/{slug}/images/
  → 详细文案（article.md）
  → [新增] LLM 推断视觉主题 → script.json.style
  → 口播脚本（script.json）+ 抖音/小红书文案
  → HTML 分镜（使用新模板 + 主题色 + 本地图片）
  → TTS + Playwright 录制 + 音视频合并 → final.mp4
```

### 新增目录结构

```
output/{slug}/
  ├── images/
  │   ├── page-screenshot.png   # 原新闻/GitHub 页面全屏截图
  │   └── img-001.jpg ~ img-N.jpg  # 从页面提取的图片
  ├── article.md
  ├── script.json               # 新增 style 字段
  ├── shot*.html
  └── ...
```

### script.json 新增字段

```json
{
  "topic": "...",
  "style": "apple",
  "social": { ... },
  "shots": [ ... ]
}
```

## 一、图片采集

### 时机

在 Step 1（搜索/理解新闻）完成后、写 article.md 之前，Agent 对主要信息来源页面执行：

1. **全页截图**：用 Playwright 对目标 URL 截取全屏快照，保存为 `images/page-screenshot.png`
2. **图片提取**：从页面 HTML 中筛选有实质内容的 `<img>`（过滤图标、广告、1px 埋点），下载前 N 张（建议 3～5 张）到 `images/img-001.jpg` 等

### Agent 选用规则

- 图片类模板（`image-caption`、`split-image`）必须引用 `images/` 里已下载的图片
- 引用时使用相对路径，如 `images/img-001.jpg`（HTML 与 images/ 同在 `output/{slug}/` 下）
- 若图片采集失败，降级为纯文字模板，不中断流程

## 二、视觉主题系统

### 四套主题

| 主题名 | 背景色 | 强调色 | 次要文字色 | 典型场景 |
|--------|--------|--------|-----------|---------|
| `apple` | #000000 | #0071e3 | #a1a1a6 | AI/科技/产品发布 |
| `cyber` | #050510 | #00ff88 | #7aff7a | 编程/开源/GitHub/极客 |
| `media` | #1a1a1a | #f59e0b | #d1d1d1 | 财经/政策/市场/社会事件 |
| `light` | #f8f8f8 | #e84393 | #555555 | 生活/消费/测评/小红书风 |

### 实现方式

所有模板通过 CSS 变量（`--bg`、`--accent`、`--muted`、`--text`）驱动配色。主题以 `data-theme` 属性注入 `<html>` 标签，Agent 生成 HTML 时写入对应值。

```html
<html data-theme="cyber">
```

```css
[data-theme="apple"] { --bg: #000; --accent: #0071e3; --muted: #a1a1a6; --text: #f5f5f7; }
[data-theme="cyber"]  { --bg: #050510; --accent: #00ff88; --muted: #7aff7a; --text: #e0ffe0; }
[data-theme="media"]  { --bg: #1a1a1a; --accent: #f59e0b; --muted: #d1d1d1; --text: #f5f5f5; }
[data-theme="light"]  { --bg: #f8f8f8; --accent: #e84393; --muted: #555; --text: #111; }
```

### 主题选择流程

1. Agent 写完 `article.md` 后，基于文案的主题性质、情感基调、目标受众，**自主判断**最合适的主题并写入 `script.json` 的 `style` 字段
2. SKILL.md 中对每个主题给出描述性说明（感觉/场合），供 Agent 参考，不做关键词规则
3. 用户可在命令中加 `--style cyber` 覆盖 Agent 的选择，generate_video.py 读取该参数并注入 HTML

## 三、模板库扩充

### 完整模板列表（13 种）

#### 原有模板（保留，升级为 CSS 变量版）

| 模板名 | 说明 |
|--------|------|
| `cover` | 封面：类别 badge + 大标题 + 副标题 |
| `features` | 功能列表：带 Lucide 图标的逐条出现列表 |
| `compare` | 多列对比卡片，中间列高亮 |
| `stats` | 大数字 + 计数动画 + 说明文字 |
| `ending` | 结尾：要点回顾 + 互动提问 |

#### 新增文字类模板

| 模板名 | 说明 | 典型用途 |
|--------|------|---------|
| `quote` | 大号引号 + 震撼一句话 + 来源署名 | 钩子镜头、名人语录 |
| `timeline` | 竖向时间轴，节点逐条出现 | 事件经过、版本迭代 |
| `pros-cons` | 两列：✓ 优点 / ✗ 缺点，逐行出现 | 产品评测、政策分析 |
| `ranking` | 1-2-3 名次，从下到上动画出现 | 排行榜、Top N 盘点 |
| `person` | 圆形头像占位 + 姓名/职位 + 一句话定位 | CEO 动态、关键人物 |
| `alert` | 全屏强调色背景 + 极大字号 + 图标 | 重磅钩子、数字冲击 |

#### 新增图片类模板

| 模板名 | 说明 | 图片来源 |
|--------|------|---------|
| `image-caption` | 图片占约 60% 高度 + 下方标题/说明文字 | `images/` 目录 |
| `split-image` | 上半图片 + 下半文字（或左右各半） | `images/` 目录 |

### Agent 选模板的原则

- 每个镜头根据该段口播内容的性质选择最匹配的模板
- 图片类模板：仅在 `images/` 目录有可用图片时使用
- `alert` 模板：仅用于钩子镜头（shot1）或强调某一数字/事件时
- 不同镜头应尽量使用不同模板，避免连续重复

## 四、文案质量提升

### 4.1 禁止说明书体（强化现有原则）

在 SKILL.md 的"文案核心原则"部分新增**逐句对照改写示例**（3 组以上），确保 Agent 有具体参照而非仅凭原则描述：

```
坏：GPT-5 带来四大升级：推理能力提升 10 倍、多模态全面增强……
好：等了整整一年，发布会结束那一刻，所有人都沉默了……
```

### 4.2 钩子自检清单

Agent 写完第一镜 narration 后，必须过以下检查：

- [ ] 这句话能在 1.5 秒内让人停住吗？
- [ ] 有没有数字、反差、或悬念中的一种？
- [ ] 有没有主语是"你"或"所有人"的代入感？

任何一项不满足 → 重写钩子，再检查。

### 4.3 TTS 节奏规范（新增）

在口播技术参数部分新增：

- 用 `……` 表示 0.5 秒停顿，在情绪词后、数字后必加
- 单句严格 ≤15 字，超出必须拆句
- 禁止用逗号连接两个完整句子（改用句号断开）
- 每段 narration 字数建议 40～80 字（对应 5～10 秒 TTS）

示例：
```
坏：GPT-5今天正式发布，推理能力提升了10倍，价格却出乎所有人的预料。
好：GPT-5……今天发布了。推理能力，提升十倍。但价格……出乎所有人意料。
```

### 4.4 社媒文案质检清单（新增）

Agent 生成 `social` 字段后，必须自检：

**抖音标题**
- [ ] 每条 ≤30 字
- [ ] 至少有一个悬念词或数字
- [ ] 不含"震惊"、"重磅"、"史上最"等平台限流词

**小红书标题**
- [ ] 含 emoji（1～3 个，不过密）
- [ ] 有痛点词或利益词

**正文通用**
- [ ] 结尾有互动引导（"你怎么看"/"评论区见"等）
- [ ] 话题标签 3～6 个，不堆砌
